Embedding the <raphael> element in the same html file

You can safely put one or more <raphael> elements in your html documents. An example on how to do that is here. Please take the time to read its sourcecode.

For portability reasons is important that the <raphael> elements declaration contains the 

xmlns="http://raphaeljs.com"
atribute.

Note: <raphael> elements can contain html attributes like id or class for locating the element on the dom and render each <raphael> individually if neccesary.

Using ajax for loading the xml file

A xml document available by ajax can be rendered in an html document like this example shows. Take the time to read the source code. It is only a matter of including the reuired .js files in your HTMl document and the following: 

<script type="text/javascript">
rm.renderAjax("drawing1.xml");
</script>

and the file drawing1.xml will be rendered in current document.

Using the XSL sylesheet for rendering the raphael .xml files directly.

You compose your raphael writings in a .xml file, like this drawing. The previous link opened in a modern browser should show you my drawing. Notice that I'm being including a xsl stylesheet called drawing1.xml using the following: 

<?xml-stylesheet type="text/xsl" href="drawing1.xsl"?>

We must also write the configuration XSL file drawing1.xsl that must contain some information, like the path to dependency javascripts (jquery, raphael, etc), the title of the html document, and the path of the raphael.xsl, the main raphael markup XSL file for transforming a .xml docuemnt with our drawing into an xhtml document viewable by the browser. So create drawing1.xsl file in the same folder as drawing1.xml and with a content like this:

<?xml version="1.0" encoding="ISO-8859-1"?>
<xsl:stylesheet version="1.0"
	xmlns:xsl="http://www.w3.org/1999/XSL/Transform">		
	
	<!-- please fix the following variables and paths -->
	<xsl:variable name="doc.title" select="'raphael markup test1'" />
	<xsl:variable name="raphael.id" select="'raphaelDocument1'" />	
	<xsl:variable name="raphael.js.path" select="'../../deps/raphael-min.js'" />
	<xsl:variable name="jquery.js.path" select="'../../deps/jquery-1.7.1.min.js'" />
	<xsl:variable name="rm.js.path" select="'../../rm.js'" />
	<xsl:include href="../../raphael.xsl"/> 	
	
	<xsl:template match="*">
	</xsl:template>
</xsl:stylesheet>

IMPORTANT: this only works in firefox and chrome and it is important, in the xml file to define the <raphael> element just without ANY attributes, like in the demo.

For this to work on IE the <raphael> element must be defined like this: 

<raphael xmlns="http://raphaeljs.com">
